Answer N. 1â€‹

https://seclists.org/fulldisclosure/2022/Jan/26

The exploit in the link above is a Microsoft Defender Bypass, which allows to disable / bypass the security measures put in place by Microsoft Windows Defender.

One of its ways of working is by the user passing an extra path traversal when referencing mshtml or using concatenation when constructing the URL scheme portion of the payload. This exploit is particularly interesting because it demonstrates how an attacker can bypass the security measures of a well-known and widely-used antivirus software.

It highlights the fact that there is no perfect antivirus / security software, and it is important to be aware of viruses and other malware that can be used to attack your computer.

https://seclists.org/fulldisclosure/2021/Oct/19

The exploit in the link above is a Trojan that creates a directory with insecure permissions under the Windows\SysWOW64 folder, granting full permissions to the Everyone user group.

THis allows the users to rename, delete or modify the files in the directory, which can be used to create a backdoor for the attacker to access the system. This exploit is made possible due to the use of insecure permissions on the malware's directory, which can be exploited by a standard user to disable or modify the malware.

This exploit highlights the importance of using secure permissions on files and directories. It also highlights the importance of using a standard user account when browsing the internet, as it is less likely to be compromised by malware.
